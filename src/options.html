Eliminate Pre-Navigation Flash

Flash Guard injects dark CSS as early as possible, but Firefox paints a white canvas before any HTTP response arrives. This single white frame is visible on very fast pages (e.g. localhost). No extension can override this because it happens before the response body exists.

One-Time Fix via about:config

Change Firefox's default canvas color to dark so the pre-navigation frame is no longer white:

  1. Open a new tab and type about:config in the address bar, then press Enter.
  2. Accept the warning if prompted.
  3. Search for browser.display.background_color
  4. Change the value from #FFFFFF to #1a1a1a

This is a global Firefox preference. It changes the default background for all pages that do not set their own background color. You can revert it at any time by resetting the value back to #FFFFFF.
